## 📖 推荐阅读路径

### 新人入职（第1天）
```
1. 先看《管理层摘要》了解项目概况          (5分钟)
2. 再看《项目路线图》了解发展规划          (10分钟)
3. 然后看《下一步计划速览》了解当前任务     (10分钟)
4. 最后看《下一步行动计划》了解详细内容     (30分钟)
```

### 每日站会前（每天早上）
```
1. 快速浏览《下一步计划速览》              (2分钟)
2. 查看《P3每周行动清单》本周任务          (3分钟)
```

### 周报准备（每周五）
```
1. 查看《项目进度甘特图》更新进度条        (5分钟)
2. 查看《进度与里程碑》总结完成情况        (10分钟)
3. 准备周报PPT/邮件                      (20分钟)
```

### 月度评审（每月末）
```
1. 查看《进度与里程碑》总结月度成果        (10分钟)
2. 查看《项目路线图》对比计划进度          (10分钟)
3. 查看《管理层摘要》准备汇报材料          (10分钟)
4. 准备评审PPT                          (30分钟)
```

## ❓ 常见问题

### Q1: 文档太多，看不过来怎么办？
**A**: 根据你的角色，只看推荐的2-3个核心文档即可。开发人员重点看"下一步计划速览"和"P3每周行动清单"。

### Q2: 文档更新频率是多少？
**A**: 
- **每日更新**: 无
- **每周更新**: 下一步计划速览、进度与里程碑、项目进度甘特图
- **每月更新**: 项目路线图、管理层摘要
- **按需更新**: 其他参考文档

### Q3: 发现文档错误或过时怎么办？
**A**: 在GitHub提Issue或直接联系项目经理。

### Q4: 需要打印哪些文档？
**A**: 
- 日常工作：打印"下一步计划速览"（1页）
- 周会汇报：打印"进度与里程碑"（2-3页）
- 月度评审：打印"管理层摘要"（1页）

### Q5: 文档是中文还是英文？
**A**: 计划类文档全部是**中文**，技术文档部分中英文混合。
